Drago - Formato SGF

Riferimenti

Il formato SGF, Smart Game Format, è il formato più usato su internet per archiviare le partite di Go. Lo potete trovare all'indirizzo Specifiche SGF.

Drago aderisce quasi completamente alle specifiche del formato SGF versione FF[4]. Una descrizione di quali siano le sue conformità e quali le differenze con le specifiche è riportata qui di seguito.

Nota che non è necessario conoscere il formato per usare Drago (come qualsiasi altro programma per la gestione dei file SGF).

Proprietà di base

Le proprietà di base danno una descrizione generale di un gioco.

Nota :

  • Le proprietà gioco, applicazione, formato file e grandezza (GM, AP, FF, SZ) sono archiviate quando si crea un file se l'opzione Nuovo file con proprietà SGF complete è attivato. Altrimenti, viene archiviata solo il valore della proprietà grandezza se il suo valore è differente da quello di base, ovvero 19.
  • La proprietà gioco (GM) fornisce un avvertimento se il suo valore non è 1. La proprietà formato file (FF) è riconosciuta ma non fa nulla: Drago si affida alla visualizzazione di proprietà sconosciute nella barra di stato.
  • Le proprietà applicazione e stile (AP e ST) possono essere modificate nel pannello di controllo Informazioni.
  • La proprietà formato file (FF) viene visualizzata ma non può essere modificata.
  • La proprietà stile cambia temporaneamente i valori delle opzioni Mostra Prossima Mossa e Mostra Mosse Alternative. Se queste opzioni non sono modificate durante la partita in corso, esse sono ripristinate quando si carica la partita successiva.
  • Se un file di gioco non ha la proprietà CA (Tipo di Caratteri - ChArset), viene usata la codifica caratteri specificata nell'opzione "Seleziona codifica caratteri di base per i file di gioco" nella scheda (Lingua). Le codifiche caratteri gestiti da Drago sono elencate in questa opzione.

Proprietà informazioni di gioco

Le proprietà delle informazioni di gioco sono usate per dare una descrizione generale di un gioco. Sono visualizzate e modificate con il comando Informazioni.

Nota:

  • Non c'è validazione quando si modifica la proprietà data (DT). Il formato dovrebbe essere AAAA-MM-GG. Vedi le specifiche per una descrizione completa.
  • Non c'è validazione quando si modifica la proprietà risultato (RE).Il formato dovrebbe essere nella forma B+1.5. Vedi le specifiche epr una descrizione completa.
  • Non c'è validazione quando si modifica la proprietà regole (RU). Vedi le specifiche per la descrizione.

Proprietà delle Note

Le proprietà delle Note sono usate per giudicare la qualità di una mossa o di una posizione:

  • Le proprietà C e N (Commento e Nome) sono modificate e visualizzate nella finestra principale.
  • Le proprietà DM, GB, GW, HO, UC, BM, DO, IT e TE sono modificate con il menu Modifica/Inserisci e visualizzano un messaggio nella barra di stato.
  • La proprietà V (Valore) non è gestita.

Proprietà delle Annotazioni

Le proprietà delle Annotazioni accettano zone rettangolari come SQ[ab:cd]. La singola proprietà annotazione (M) è obsoleta; è visualizzata (come l'annotazione triangolo) ma non può essere inserita con Drago. La proprietà etichetta (LB) è visualizzata, ma limitatamente a 3 caratteri. Il comando Modifica/Annotazioni/Numeri è usato per aggiungere il numero della mossa come farebbe la proprietà LB per il nodo.

Proprietà Impostazioni

Le proprietà delle impostazioni AB, AE e AW accettano zone rettangolari come AB[ab:cd].
